# IAToolkit Benchmark Testing Guide

## Overview — Bring Your Own Data (BYOD)

IAToolkit's benchmark runner lets you evaluate any prompt/model combination **entirely with your own private data**.
Your dataset never leaves your infrastructure: the runner reads from a local `.xlsx` file, calls the agent pipeline, and writes results back to a local output file.

This guide covers:
1. [Dataset format](#1-dataset-format)
2. [Running the benchmark](#2-running-the-benchmark)
3. [Understanding the output](#3-understanding-the-output)

## 1. Dataset Format

Your input file **must** be an `.xlsx` spreadsheet containing the following columns (in any order):

| Column | Type | Required | Description |
|---|---|---|---|
| `username` | string | ✅ | IAToolkit user identity that executes each query (must exist in the DB) |
| `client_identity` | string | ✅ | The client RUT / identifier passed to the agent as context |
| `prompt_name` | string | ✅ | Name of the registered prompt to test |
| `question` | string | ✅ | The natural-language question sent to the agent |
| `model` | string | ✅ | LLM model identifier (e.g. `gpt-4o`, `claude-3-5-sonnet-20241022`, `gemini-2.0-flash`) |

> **Tip:** Extra columns you add are preserved untouched in the output file — use them for notes, expected answers, or test IDs.

A ready-to-use template can be generated with:

```bash
python examples/generate_benchmark_template.py
```

## 2. Running the Benchmark

Make sure your Flask application is fully configured (`.env` loaded, database accessible).

```bash
flask run-benchmark <company_short_name> <path_to_dataset.xlsx>
```

**Example:**

```bash
flask run-benchmark acme_corp ./benchmarks/acme_q1_2025.xlsx
```

You can also check all available options:

```bash
flask run-benchmark --help
```

**What happens internally:**

1. The runner loads the `.xlsx` file into a DataFrame.
2. For each row it calls `query_service.llm_init_context()` and then `query_service.llm_query()`.
3. Timing and token statistics are collected for every row.
4. A results file is written alongside the input file (see below).

## 3. Understanding the Output

The runner writes `<original_filename>_results.xlsx` to the same directory as the input file.
The output file contains all your original columns **plus** these result columns:

| Column | Description |
|---|---|
| `status` | `OK` — query completed successfully · `FAILED` — an error occurred |
| `context_time` | Time (seconds) to initialise the LLM context for this row |
| `gpt_time` | Time (seconds) the LLM took to return an answer |
| `answer` | The agent's response text |
| `error_message` | Human-readable error detail when `status = FAILED` (max 512 chars) |
| `query_id` | Internal query ID — useful for cross-referencing with the database |
| `in_tokens` | Input tokens consumed by the LLM |
| `out_tokens` | Output tokens generated by the LLM |
| `retry` | Number of SQL retries performed (0 = first-attempt success) |

**Example output path:**

```
./benchmarks/acme_q1_2025_results.xlsx   ← written automatically
./benchmarks/acme_q1_2025.xlsx           ← your original input (unchanged)
```

## 4. Security Best Practices

- **Keep datasets outside the repository.** Store your `.xlsx` files in a directory that is not tracked by git (e.g. `~/benchmarks/` or a mounted network drive).
- **Add result files to `.gitignore`** to prevent accidental commits:

  ```gitignore
  # Benchmark data — never commit proprietary datasets
  benchmarks/
  *_results.xlsx
  ```

- The benchmark runner **does not upload** your dataset to any external service. All LLM calls go through the same proxy/gateway already configured for your company.
- Rotate API keys used during benchmarking if the keys have access to production data.

## 5. Troubleshooting

| Symptom | Cause | Fix |
|---|---|---|
| `❌ Benchmark failed: solo se leer archivos .xlsx` | Input file is not `.xlsx` | Convert your file to `.xlsx` format |
| `❌ Benchmark failed: El archivo no fue encontrado en:` | Path is wrong or file does not exist | Double-check the path; use an absolute path if in doubt |
| `❌ Benchmark failed: La planilla debe contener las columnas:` | Missing required column(s) | Add all 5 required columns (see [Dataset Format](#1-dataset-format)) |
| `❌ Benchmark failed: Compañía X no encontrada.` | `company_short_name` is not registered | Check `app.py` → `register_company(...)` calls |
| All rows have `status = FAILED` | User or prompt not found in DB | Ensure `username` exists and `prompt_name` is registered for this company |
